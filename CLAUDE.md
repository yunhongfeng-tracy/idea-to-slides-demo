# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

"想法变成图片"——将用户的想法、笔记、分析等文字内容，通过白板手绘风格的可视化图片来解释和呈现。核心流程：

```
用户结构化输入 → 快速确认（如有遗漏） → 内容分析 + 提示词生成 → 生成图片
```

## 用户输入模板

用户通过以下模板一次性提供关键信息，减少来回澄清轮次：

```
【想法】：（核心想法，自由描述）
【角色】：（涉及哪些角色/对象，它们之间什么关系）
【重点】：（最想让看图的人记住什么）
【补充】：（容易遗漏的细节，没有就不填）
```

收到模板后的处理流程：
1. **快速确认**：如果信息充分，直接进入下一步；如果有明显遗漏，最多追问 1-2 个问题
2. **保存想法梳理 + 图类型分析 + 提示词**到 `doc/XX-场景名.md`（合并为一个文件）
3. **直接生成图片**，保存到 `py/XX-场景名-图类型.png`
4. 用户看图反馈，按需微调

目标：**1-2 轮对话完成一张图**。

### 澄清原则

- 用户使用了输入模板且信息充分时，**跳过澄清，直接出图**
- 仅在关键信息明显缺失时追问，**最多 1-2 个问题**
- 如果用户没用模板（自由输入），则快速提炼后确认一次，不逐条追问

## 图类型自动选择

根据内容特征自动选择最合适的白板图类型，**不需要让用户选**，直接出最合适的。用户不满意再换。

| 内容特征 | 图类型 | 适用场景 |
|----------|--------|----------|
| 有新旧对比、范式转变 | **对比型**（Before vs After） | "从X变成Y"、优劣对比 |
| 有多角色交互、系统架构 | **架构型**（Hub & Spoke） | 平台、多方协作、系统关系 |
| 有时间演进、迭代过程 | **时间线型**（Timeline） | 发展历程、迭代优化、阶段性变化 |
| 有线性流转、输入→输出 | **流程型**（Flow） | 工作流、数据流、交互链路 |
| 有核心概念需展开解释 | **中心辐射型**（Mind Map） | 一个概念的多维度展开 |

## 常用命令

```bash
# 生成图片
python -X utf8 py/generate_image.py --prompt "提示词" --output "py/输出文件.png" --aspect-ratio 16:9 --image-size 4K

# 从文件读提示词
python -X utf8 py/generate_image.py --prompt-file prompts/example.txt --output images/example.png
```

## API 配置

API Key 存放在项目根目录 `api.md` 文件中。支持纯 key 格式（一行即 key）或 `GRSAI_API_KEY=xxx` 格式。脚本会自动向上搜索 `api.md` 或 `api` 文件。优先级：命令行参数 > 环境变量 > api 文件。

## 提示词编写规范

### 风格底板（每次复用，不用重写）

所有白板图提示词以此为基础，追加在内容描述之后：

```
Style: clean white whiteboard background, hand-drawn sketch like Excalidraw, thick colorful marker strokes, casual handwritten Chinese labels, simple doodle icons, wobbly hand-drawn arrows. No photorealistic elements.
```

### 内容编写规则

- 提示词主体用英文（AI 绘图模型对英文效果更好）
- 开头加 `ALL TEXT MUST BE IN CHINESE`，所有标签直接写中文
- 英文例外：仅专有名词/品牌名保留英文
- 关键中文词在提示词中重复出现，防止 AI 生成时写错字
- 用不同颜色区分不同角色或阶段（如蓝色=用户侧，绿色=正面，红色=问题）
- 推荐参数：`--aspect-ratio 16:9 --image-size 4K`

## 工作流程（每个场景两份产出）

每个场景保留两份资料：

1. **文档**（`doc/XX-场景名.md`）：想法梳理 + 图类型分析 + 完整提示词 + 生成参数
2. **图片**（`py/XX-场景名-图类型.png`）：生成的白板图

## PPT 测试用例生成流程

用于为不同 PPT 工具生成测试用例。起点是一句话主题，经过 4 个阶段逐步展开，最终产出每页的图片生成提示词。

### 四阶段流水线

```
一句话主题 → 大纲 → 完整文档 → 分页内容 → 每页布局提示词
```

| 阶段 | 文件 | 说明 |
|------|------|------|
| Step 1 大纲 | `step1-大纲.md` | 从一句话提炼出结构化大纲（章节 + 要点） |
| Step 2 完整文档 | `step2-完整文档.md` | 将大纲丰富为完整内容（数据、案例、细节） |
| Step 3 分页内容 | `step3-分页内容.md` | 按 PPT 页拆分（封面 + 正文 + 总结），每页含标题、要点、视觉建议 |
| Step 4 布局提示词 | `step4-布局提示词.md` | 每页生成白板手绘风格的图片提示词，含布局区域、颜色、图标描述 |

### 文件组织

测试用例存放在 `test-cases/主题名/` 目录下，每个主题一个文件夹：

```
test-cases/
└── 介绍深圳/
    ├── step1-大纲.md
    ├── step2-完整文档.md
    ├── step3-分页内容.md
    └── step4-布局提示词.md
```

### Step 4 提示词格式规范

每页提示词包含以下结构（与项目已有的 `PPT分享提示词2/` 格式一致）：

- **图类型**：根据内容自动选择（封面型、中心辐射型、时间线型、分区展示型、流程型、关键词云型等）
- **通用约束**：中文文字、无英文、无写实元素、无白板边框/笔
- **通用风格**：白板手绘、Excalidraw 风格、彩色马克笔、站酷快乐体字体
- **提示词正文**：按区域描述布局（标题区域 → 内容区域 → 底部区域），每区域标注占比、颜色、图标、文字内容
- **生成参数**：16:9 宽高比、4K 尺寸

## 架构

- `py/generate_image.py`：核心脚本，调用 Nano Banana Pro API（GRSAI）生成图片。内置 HTTP 自动重试机制（5 次重试，指数退避）应对 SSL/网络波动。
- `SKILL.md`：Skill 文档，描述脚本用法和完整工作流。
- `api.md`：API Key 存储文件（不要提交到公开仓库）。
- `doc/`：场景文档（想法梳理 + 提示词，每场景一个文件）。
- `py/`：脚本和生成的图片。
- `test-cases/`：PPT 测试用例（每个主题一个文件夹，含 4 阶段产出文件）。
